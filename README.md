
# TMC Policy Assistant

An internal HR chatbot for **TallyMarks Consulting (TMC)**. Employees ask questions about
company policy — leave, dress code, travel, loans, maternity, TADA, ethics, OPD, business
conduct — and get answers grounded strictly in TMC's official policy PDFs, with citations
back to the source document and page.

Retrieval-Augmented Generation (RAG) backend in FastAPI + LangChain, with a React frontend.

## How it works

```
TMC_Data/*.pdf
      │  ingest.py (chunk + embed)
      ▼
faiss_index/                       (local vector store)
      │
      ▼
rag_chain.py
  1. history-aware retriever   → rewrites follow-up questions into standalone queries
  2. stuff-documents chain     → answers ONLY from retrieved chunks, cites source + page
  3. RunnableWithMessageHistory → one persisted conversation thread per session_id
      │
      ▼
main.py (FastAPI)  ──HTTP/SSE──►  React frontend
      │
      ▼
sessions.db (SQLite, via session_store.py)
```

- **LLM**: Groq (`ChatGroq`)
- **Embeddings**: HuggingFace (`langchain_huggingface`)
- **Vector store**: FAISS, persisted to disk in `faiss_index/`
- **Conversation memory**: SQLite (`sessions.db`), keyed by `session_id`
- **Streaming**: token-by-token via Server-Sent Events (`/ask/stream`)

## Features

- Answers grounded strictly in the ingested PDFs — explicitly refuses to guess or use outside knowledge
- Inline citations: `(Source: <file>, page <page>)`, surfaced in the UI as an expandable "Sources" panel
- Multi-turn conversation memory per session, with follow-up question rewriting
- Streaming responses (SSE) for a live, token-by-token typing effect
- Multiple isolated conversations per user, listed and switchable in the sidebar
- Retry/backoff on transient Groq rate limits (429s)
- Retrieval & faithfulness evaluation scripts (`eval/`)

## Prerequisites

- Python 3.10+
- Node.js 18+
- A [Groq API key](https://console.groq.com)

## Setup

### 1. Backend

```bash
cd backend
python -m venv .venv
.venv\Scripts\activate        # Windows
# source .venv/bin/activate   # macOS/Linux

pip install -r requirements.txt
```

Create `backend/.env`:

```ini
GROQ_API_KEY=your_groq_api_key_here
GROQ_MODEL=llama-3.3-70b-versatile
EMBEDDING_MODEL=sentence-transformers/all-MiniLM-L6-v2
FRONTEND_ORIGIN=http://localhost:5173
TOP_K=4
```

> Adjust keys to match whatever `config.py` actually reads — the above are the ones referenced
> in `rag_chain.py` and `main.py`.

Drop your policy PDFs into `backend/TMC_Data/`, then build the vector store:

```bash
python ingest.py
```

Start the API:

```bash
uvicorn main:app --reload --port 8000
```

Verify it's up: [http://127.0.0.1:8000/docs](http://127.0.0.1:8000/docs)

### 2. Frontend

```bash
cd frontend
npm install
cp .env.example .env   # set VITE_BACKEND_URL if not http://localhost:8000
npm run dev
```

Open the printed local URL (default [http://localhost:5173](http://localhost:5173)).

## API reference

| Method | Path                    | Description                                                   |
|--------|--------------------------|-----------------------------------------------------------------|
| GET    | `/health`                | Liveness + whether the vector store is built                    |
| POST   | `/ask`                   | `{question, session_id}` → `{answer, citations, session_id}`     |
| POST   | `/ask/stream`            | Same input, streamed as SSE (`token` / `citations` / `error` / `done`) |
| GET    | `/history/{session_id}`  | Full message history for a session                              |
| DELETE | `/history/{session_id}`  | Clears a session's history                                      |
| GET    | `/sessions`              | List of past conversations                                      |

## Evaluation

```bash
cd backend
python eval/eval_retrieval.py     # retriever quality against eval/questions.csv
python eval/eval_faithfulness.py  # checks answers are grounded in retrieved context
```

## Notes

- `faiss_index/`, `sessions.db`, `.env`, and `__pycache__/` should stay out of version control —
  see `.gitignore` below.
- Re-run `ingest.py` whenever `TMC_Data/` changes, then restart the backend.
